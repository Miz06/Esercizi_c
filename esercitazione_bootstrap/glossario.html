Socket:

Un socket è un endpoint per la comunicazione tra due macchine. Può essere utilizzato per inviare e ricevere dati attraverso una rete.

TCP (Transmission Control Protocol):

Un protocollo di comunicazione che garantisce la consegna affidabile e ordinata dei dati tra due dispositivi.

UDP (User Datagram Protocol):

Un protocollo di comunicazione che consente l'invio di messaggi senza stabilire una connessione, non garantendo l'ordine o l'affidabilità.

OSI (Open Systems Interconnection):

Un modello di riferimento per la comunicazione di sistemi informatici che definisce sette livelli (fisico, data link, rete, trasporto, sessione, presentazione, applicazione) per standardizzare la comunicazione tra i dispositivi.

Three-way handshake:

Un processo di inizializzazione di una connessione TCP tra client e server che avviene in tre fasi: SYN, SYN-ACK, ACK.

Endpoint:

Un punto di comunicazione in una rete, rappresentato da un indirizzo IP e una porta, che consente l'invio e la ricezione di dati.

DNS (Domain Name System):

Un sistema di nomi di dominio che traduce i nomi di dominio (come www.example.com) in indirizzi IP utilizzabili da computer e dispositivi.

Client:

Un dispositivo o programma che invia richieste a un server per ottenere servizi o risorse.

Server:

Un dispositivo o programma che fornisce servizi o risorse a un client attraverso una rete.

socket():

Una funzione che crea un nuovo socket per la comunicazione di rete tra un client e un server.

connect():

Una funzione utilizzata dal client per stabilire una connessione con un server tramite un socket.

bind():

Una funzione che associa un socket a un indirizzo IP e a una porta specifici, permettendo al server di ascoltare le richieste.

listen():

Una funzione che mette il server in attesa di connessioni da parte dei client.

accept():

Una funzione utilizzata dal server per accettare una connessione in ingresso da un client.

write():

Una funzione che consente di inviare dati attraverso un socket, dal client al server o viceversa.

read():

Una funzione che consente di leggere i dati ricevuti attraverso un socket, dal server al client o viceversa.

recv():

Una funzione simile a read(), utilizzata per ricevere dati attraverso un socket, ma con la possibilità di specificare la dimensione del buffer di lettura.

send():

Una funzione che consente di inviare dati attraverso un socket, dal client al server o viceversa, permettendo di specificare un buffer per i dati inviati.

close():

Una funzione che chiude una connessione socket, terminando la comunicazione tra client e server.
